# Search Tool

A TF-IDF powered web search engine built in Python. Crawls a target website, builds an inverted index with relevance scoring, and provides ranked search results via a command-line interface.

Built as part of a Web Data and Services coursework assessment at the University of Leeds.

---

## Architecture Overview

```
search-tool/

│   ├── crawler.py    # BFS web crawler with politeness controls and URL normalisation
```

### Key Design Decisions

| Component | Choice | Rationale |
|-----------|--------|-----------|
| Index structure | `dict[str, dict[str, dict]]` | O(1) average lookup for both term and document |
| Ranking | TF-IDF | Down-weights common words; up-weights distinctive terms |
| Serialisation | JSON | Human-readable, portable, no binary dependencies |
| Traversal | BFS via deque | O(1) pops; visits pages level-by-level without deep recursion |
| Stop words | Custom frozenset | frozenset gives O(1) membership test vs O(n) for a list |
| URL normalisation | Trailing slash stripping | Prevents the same page being indexed twice e.g. /page and /page/ |
| Token cleaning | Strip leading/trailing apostrophes | Prevents quoted HTML text producing noise tokens like `'beautiful'` |

### TF-IDF Formula

```
TF  = count(term, doc) / total_terms(doc)
IDF = log((N + 1) / (df + 1)) + 1      # smoothed to avoid division by zero
TF-IDF = TF × IDF
```

Documents are ranked by the **sum of TF-IDF scores** for all matched query terms. This means a page where a term is rare and frequent scores higher than one where the term is common across all pages.

## Usage

All commands are run from the project root.

### `build` — Crawl and index a website

```bash
python src/main.py build
```

Options:

| Flag | Default | Description |
|------|---------|-------------|
| `--url URL` | `https://quotes.toscrape.com` | Seed URL to crawl |
| `--output PATH` | `data/index.json` | Where to save the index |
| `--max-pages N` | unlimited | Cap crawl at N pages |

Examples:
```bash
# Full crawl of the default site
python src/main.py build

# Quick test crawl (5 pages only)
python src/main.py build --max-pages 5

# Crawl a different site
python src/main.py build --url https://example.com --output data/example.json
```

The crawler respects a **6 second politeness delay** between requests and handles HTTP errors, timeouts, and connection failures gracefully without crashing.

---

### `load` — Validate an existing index

```bash
python src/main.py load
python src/main.py load --index data/index.json
```

Confirms the index file exists and is valid, then prints summary statistics.

---

### `print` — Display index contents

```bash
python src/main.py print
python src/main.py print --max-terms 100
```

Options:

| Flag | Default | Description |
|------|---------|-------------|
| `--index PATH` | `data/index.json` | Index file to display |
| `--max-terms N` | `50` | Max number of terms to show |

---

### `find` — Search the index

```bash
python src/main.py find love
python src/main.py find "love truth"
python src/main.py find "love AND truth"
python src/main.py find "love OR hate" --top 5
```

Options:

| Flag | Default | Description |
|------|---------|-------------|
| `--index PATH` | `data/index.json` | Index file to search |
| `--top N` | `10` | Number of results to return |

**Boolean operators:**
- `love truth` → OR search (pages containing *any* term)
- `love AND truth` → pages containing *both* terms
- `love OR truth` → same as default OR

**Edge cases handled:**
- Empty query → friendly message
- Stop-word-only query → no results message
- Unknown word → no results message
- AND with no intersection → no results message

## Complexity Analysis

| Operation | Time Complexity | Space Complexity |
|-----------|----------------|-----------------|
| Crawl P pages | O(P × L) | O(P) |
| Add document | O(T) | O(T) |
| Compute TF-IDF | O(T × D) | O(1) |
| Search OR | O(k × D) | O(R) |
| Search AND | O(k × D) | O(R) |
| URL deduplication | O(1) per URL | O(P) |

Where: P = pages, L = links per page, T = tokens per doc, D = docs per term, k = query terms, R = results.

The choice of a hash-based dict for the index means term lookup is O(1) average case, significantly faster than a list-based approach which would require O(n) scanning per query.
